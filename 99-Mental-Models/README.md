# Engineering Notes

> *"Don't document what you memorized. Document what you finally understood."*

Welcome to my Engineering Notes.

This repository is the central knowledge base for my journey developing as a Systems Administrator, Network Engineer, and DevOps Engineer.

Rather than collecting commands, this repository documents the concepts, mental models, troubleshooting processes, and hands-on projects that have helped me build a deeper understanding of technology.

Everything in this repository is based on:

- Hands-on home lab experience
- Real-world troubleshooting
- Personal software projects
- Experimentation
- Reflection
- Continuous learning

---

# Learning Philosophy

I learn best by building.

Instead of memorizing commands, I prefer to:

1. Plan the problem.
2. Build the solution.
3. Debrief what happened.
4. Document what I learned.
5. Connect the concept to other technologies.
6. Revisit it later to reinforce understanding.

My goal is not simply to know *what* to type, but to understand *why* systems behave the way they do.

---

# Engineering Principles

- Understand before memorizing.
- Experiment before explanation.
- Build before reading.
- Verify assumptions.
- Automate repetitive work.
- Document lessons learned.
- Continuously improve.
